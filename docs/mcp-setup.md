# Подключение MCP (работа через ИИ)

MCP — протокол, которым Claude подключается к внешним инструментам. Через
него плагин открывает Claude доступ к вашей модели Revit: посмотреть
помещения, светильники, щиты, цепи — а с вашего разрешения ещё и создавать
или менять элементы. Что можно попросить и как это выглядит на практике —
в [docs/ai-assistant-usage.md](ai-assistant-usage.md). Здесь — только
подключение, займёт пару минут.

## Что понадобится

- Установленный плагин ([инструкция](installation.md)) — MCP-сервер идёт в
  комплекте, отдельно скачивать ничего не нужно
- .NET Runtime 8 (x64) — [скачать](https://dotnet.microsoft.com/download/dotnet/8.0)
  (нужен именно Runtime, не SDK)
- Claude Desktop или Claude Code

## Claude Desktop

Откройте (или создайте) файл `%APPDATA%\Claude\claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "revit-erp": {
      "command": "C:\\Users\\<пользователь>\\AppData\\Roaming\\Autodesk\\Revit\\Addins\\2024\\ElectricityRevitPlugin\\run-mcp.cmd"
    }
  }
}
```

Подставьте своё имя пользователя Windows и год версии Revit. Путь до
`run-mcp.cmd` не меняется при обновлениях плагина — прописали один раз и
забыли. Сохраните файл и перезапустите Claude Desktop.

## Claude Code

Тот же принцип, только в файле `.mcp.json` в корне проекта (или командой
`claude mcp add`):

```json
{
  "mcpServers": {
    "revit-erp": {
      "command": "C:\\Users\\<пользователь>\\AppData\\Roaming\\Autodesk\\Revit\\Addins\\2024\\ElectricityRevitPlugin\\run-mcp.cmd"
    }
  }
}
```

## Порты и адреса настраивать не нужно

Плагин сам находит запущенный Revit и подключается к нему — никаких портов
руками. Если открыто несколько Revit сразу, просто скажите Claude «покажи
список открытых Revit» и «переключись на второй» — дальше он разберётся сам,
без перезапусков.

## Вход в систему

При первом обращении к инструментам Claude покажет ссылку и короткий код —
перейдите по ссылке в браузере, введите код, подтвердите. Дальше вход
запоминается, второй раз логиниться не придётся.

## Проверка

Спросите Claude: **«Проверь связь с Revit»**. В ответ — версия Revit,
открытый документ и подтверждение, что всё работает. Не сработало — смотрите
ниже.

## Если не заработало

| Симптом | Что проверить |
|---|---|
| Инструменты не появились у Claude | Путь к `run-mcp.cmd` в конфиге верный? Claude Desktop перезапущен после изменения конфига? |
| «Не удаётся подключиться к Revit» | Revit запущен, проект открыт, плагин виден на ленте? |
| Ошибка запуска MCP-сервера | Установлен .NET Runtime 8 (x64)? |
| Запросы уходят не в тот Revit | Открыто несколько окон — попросите Claude показать список и переключиться |
| Просит войти заново каждый раз | Убедитесь, что дошли до конца по ссылке и подтвердили вход в браузере |
