# Подключение MCP (работа через ИИ)

MCP (Model Context Protocol) — это протокол, по которому Claude подключается
к внешним инструментам. Плагин вместе с MCP даёт Claude «руки» — доступ к
вашей модели Revit: просмотр помещений, светильников, щитов и цепей, создание
и подключение элементов. Сама методика проектирования (что и в каком порядке
делать) описана отдельно — [docs/ai-assistant-usage.md](ai-assistant-usage.md).

## Требования

- Плагин ElectricityRevitPlugin уже установлен ([инструкция](installation.md)) —
  MCP-сервер устанавливается вместе с ним, отдельно скачивать ничего не нужно.
- Установлен **.NET Runtime 8 (x64)** — без него MCP-сервер не запустится.
  Скачать: [dotnet.microsoft.com/download/dotnet/8.0](https://dotnet.microsoft.com/download/dotnet/8.0)
  (нужен именно Runtime, не SDK).
- Установлен Claude Desktop или Claude Code.

## Подключение к Claude Desktop

Откройте файл `%APPDATA%\Claude\claude_desktop_config.json` (если файла нет —
создайте) и добавьте:

```json
{
  "mcpServers": {
    "revit-erp": {
      "command": "C:\\Users\\<пользователь>\\AppData\\Roaming\\Autodesk\\Revit\\Addins\\2024\\ElectricityRevitPlugin\\run-mcp.cmd"
    }
  }
}
```

Замените `<пользователь>` на имя своей учётной записи Windows, а `2024` — на
год установленной версии Revit. Путь до `run-mcp.cmd` постоянный: он не
меняется при автообновлении плагина, поэтому конфиг нужно прописать один раз.

После сохранения файла перезапустите Claude Desktop.

## Подключение к Claude Code

Аналогично, только в файле `.mcp.json` (в корне проекта, с которым работаете,
или через команду `claude mcp add`):

```json
{
  "mcpServers": {
    "revit-erp": {
      "command": "C:\\Users\\<пользователь>\\AppData\\Roaming\\Autodesk\\Revit\\Addins\\2024\\ElectricityRevitPlugin\\run-mcp.cmd"
    }
  }
}
```

## Как это работает

Revit → MCP-сервер, встроенный в плагин, ничего дополнительно запускать не
нужно — он поднимается автоматически вместе с Revit. При старте Revit плагин
сам находит свободный сетевой порт и записывает информацию о себе на диск;
Claude находит этот адрес самостоятельно. **Настраивать порты вручную не
нужно.**

Если открыто несколько окон Revit одновременно — по умолчанию Claude
обращается к последнему запущенному. Чтобы переключиться на другой, просто
попросите: «покажи список открытых Revit» и «переключись на второй» —
Claude сделает это сам, без перезапуска чего-либо.

## Авторизация

При первом обращении к инструментам плагин попросит войти в учётную запись:
Claude покажет ссылку и короткий код. Перейдите по ссылке в браузере, введите
код, подтвердите вход — и скажите Claude продолжить. Дальше вход
запоминается, повторно логиниться не нужно.

## Проверка подключения

Напишите Claude: **«Проверь связь с Revit»**. В ответ вы должны увидеть версию
Revit, открытый документ и что сервер работает. Если что-то не так — смотрите
раздел ниже.

## Если не работает

| Симптом | Что проверить |
|---|---|
| Инструменты не появились у Claude | Путь к `run-mcp.cmd` в конфиге правильный? Перезапустили Claude Desktop после изменения конфига? |
| «Не удаётся подключиться к Revit» | Revit запущен и открыт проект? Плагин ElectricityRevitPlugin установлен и загрузился (виден на ленте)? |
| Ошибка запуска MCP-сервера | Установлен .NET Runtime 8 (x64)? |
| Запросы уходят не в тот Revit | Открыто несколько Revit — попросите Claude показать список и переключиться |
| Просит войти заново каждый раз | Проверьте, что дошли до конца ссылки для входа и получили подтверждение в браузере |
