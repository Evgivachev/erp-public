# Работа через ИИ-ассистента

Прежде чем начать, подключите MCP — см. [docs/mcp-setup.md](mcp-setup.md),
это пара минут.

## Инструменты и методика — две разные вещи

MCP даёт Claude «руки» — десятки отдельных операций с моделью: посмотреть
помещения, создать светильник, найти щит, посчитать нагрузку цепи. Но сам по
себе это просто набор кнопок для ИИ — без методики Claude не знает, в каком
порядке их применять, какие нормы соблюдать и когда остановиться и спросить
вас.

Методику даёт **skill** — готовая инструкция для Claude: порядок шагов,
пороги, чек-пойнты для подтверждения. В этом репозитории уже лежит такой —
[`skills/eom-vibe-design`](../skills/eom-vibe-design), методика
вайб-проектирования электроосвещения. Установили один раз — дальше просто
описываете задачу словами, а Claude сам ведёт процесс по шагам.

## Установка skill'а

### Claude Code

Скопируйте папку `skills/eom-vibe-design` из этого репозитория в
`~/.claude/skills/eom-vibe-design/` — Claude Code подключит его
автоматически, когда запрос совпадёт по смыслу, либо вызовите явно командой
`/eom-vibe-design`.

### Claude Desktop

1. Скачайте папку `skills/eom-vibe-design` из этого репозитория и запакуйте
   её в zip-архив (внутри архива должна быть папка `eom-vibe-design`, а не
   файлы россыпью). В PowerShell это делается так:

   ```powershell
   tar.exe -a -cf eom-vibe-design.zip eom-vibe-design
   ```

   > Не используйте `Compress-Archive` из встроенного PowerShell 5.1 — он
   > пишет пути в архиве с обратными слэшами, и Claude Desktop отклоняет
   > такой файл с ошибкой «Zip file contains path with invalid characters».

2. В Claude Desktop откройте **Settings → Capabilities → Skills → Upload
   skill** и выберите получившийся zip.
3. Включите skill переключателем. Дальше он подключается автоматически, по
   смыслу вашего запроса — специальных команд не нужно.

## Как устроена методика (5 этапов)

| Этап | Что происходит |
|---|---|
| 1. Анализ | Claude изучает помещения: границы, площадь, высоту потолка, назначение — и определяет нормируемую освещённость |
| 2. Расстановка | Подбирает тип светильника, считает нужное количество, предлагает равномерную раскладку — и ждёт вашего подтверждения перед созданием |
| 3. Выключатели | Расставляет выключатели у дверей, привязывает к светильникам |
| 4. Подключение к щиту | Находит подходящий щит, показывает его текущие цепи и нагрузки, предлагает лучший вариант подключения (или новую цепь) — и снова ждёт подтверждения |
| 5. Проверка | Делает скриншот результата, повторно проверяет нагрузки цепей, отчитывается, что сделано |

Полный список используемых инструментов —
[skills/eom-vibe-design/references/tools.md](../skills/eom-vibe-design/references/tools.md);
таблица норм освещённости —
[skills/eom-vibe-design/references/illuminance-norms.md](../skills/eom-vibe-design/references/illuminance-norms.md).

## Примеры запросов

Разовая задача:

> Найди щиты освещения на уровне «Level 1» рядом с осями А-Б.

> Покажи, что подключено к щиту ЩО-1.1: цепи, нагрузки, номиналы, свободен ли
> запас.

> Эти 12 светильников (выделены в Revit) подключи к ЩО-1.1 — сначала покажи
> рекомендацию.

Полный цикл (Claude сам проведёт по всем этапам):

> Спроектируй освещение в помещениях 101–105: светильники 595×595,
> освещённость по назначению помещений, выключатели у дверей, всё подключить
> к ближайшему ЩО. Изменения модели — только после моего подтверждения на
> каждом этапе.

Проверка чужой модели, без изменений:

> Пройдись по всем цепям щита ЩО-2: где нагрузка больше 80 % номинала или
> больше 20 светильников в группе? Ничего не меняй, только отчёт.

## Что можно не бояться

- **Модель не изменится без вашего «да».** Claude сначала покажет план — что,
  сколько, куда — и только после подтверждения создаст или изменит элементы.
  Просмотр модели подтверждения не требует.
- **Любую правку можно отменить** — изменения идут через обычные транзакции
  Revit, откатываются штатным `Ctrl+Z`, как всё остальное в программе.
- Работает с одним открытым проектом за раз — если ведёте диалог с Claude,
  держите открытым тот Revit, с которым говорите.

Единицы измерения в отчётах Claude: координаты и расстояния — в футах
(так их видит Revit API), нагрузки — в ВА, номиналы автоматов — в А,
расстояние в рекомендациях по подключению — в метрах.
